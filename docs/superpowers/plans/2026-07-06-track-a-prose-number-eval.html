Goal Ship ENH-10: an LLM-as-judge for prose-embedded numbers, driven by Promptfoo, gated in CI over a frozen corpus that proves the judge itself.
Stack Python (extractor + test) · Node/npx Promptfoo · a Claude judge model · GitHub Actions.
Four units, four phases:
P0 scaffold→ P1 figures_extractor→ P2 fixture corpus→ P3 judge + promptfoo→ P4 CI gate
Offline regression layer, run on prompt/model change — the runtime validate → repair loop stays the per-run guard for structured values.

Global constraints

  • The judge is a real LLM call. Fixtures-first freezes the eval's inputs, not the judge — running the suite needs a model API key. Do not build it as "no API key needed."
  • The 1:1 mapping is load-bearing: one fixture ↔ one Promptfoo test case ↔ one judge call ↔ one slot instance. Never per-proposal, never per-fund-type.
  • figures_extractor reuses verified state as ground truth — it does not re-verify it. State correctness is owned by test_proposal_validation.py.
  • Offline-only over tracked outputs. Nothing here regenerates a committed example. The extractor reads a ConsultantState from a fixture or a throwaway fake-LLM run.
  • The buried-error fixtures are the acceptance crux (P2 + P4). If the judge can't catch them, the holistic no-extraction bet is falsified — the fallback is stated, not silently pinned away.
  • HTML companion regenerated whenever this markdown changes. Conventional commits, commit after every green phase.

  • Root make eval-prose-numbers (or npm script) that runs the eval.

  • Placeholder promptfooconfig.yaml with zero tests so npx promptfoo eval exits cleanly.
Done when: make eval-prose-numbers runs to a clean exit with no tests defined; README documents the local run + API-key requirement. Committed.

Phase 1 — figures_extractor

A pure, unit-tested state → figures function exposing only the judge-relevant deterministic numbers.
  • Create the extractor at consultant_engine/evals/figures_extractor.py (decided — engine-coupled Python lives with the package for frictionless imports; add consultant_engine/evals/__init__.py). The file header notes the language-agnostic harness lives under evals/prose_numbers/, joined by the frozen fixture JSON.
  • Define the exact figures schema (flat dict): per-fund cfs, rank, weighted_alpha, allocation_pct, per-fund exposure_pct; portfolio-level weighted aggregates — pre-computed on purpose (spec §4a): per-sleeve + portfolio weighted-avg alpha, weighted CFS, asset-class/geo exposure totals, and the count/share of funds beating benchmark. These are the derivations a consultant states in prose, so emitting them means the judge checks a prose number for membership against a ready value instead of doing the arithmetic itself (the least reliable thing to delegate to an LLM). Adding a derived field here is cheaper/safer than trusting the judge to compute it.
  • Implement extract_figures(state) -> dict — pure, no I/O; read from cfs_scores, portfolio, exposure compute; compute the derived aggregates deterministically here.
  • Write tests/evals/test_figures_extractor.py: constructed ConsultantState → assert schema fields + values including the derived aggregates (verify the averages/shares against hand-computed numbers); assert purity (no input mutation).
  • Docstring states explicitly: reuses verified state as ground truth; does not re-verify against the workbook.
Done when: CONSULTANT_ENGINE_FAKE_LLM=1 .venv/bin/python -m pytest tests/evals/test_figures_extractor.py -v green; schema fields are exactly those a prose judge could be checked against. Committed.

Phase 2 — Fixture corpus

A frozen, hand-authored corpus that proves the judge across all three categories — the buried-error set is the crux.
  • Document the JSON schema: { slot_key, figures, prose, expect, offending_sentence?, category }.
  • good — real correct pairs across why.*/watch.*/macro.impact, several funds. Mandatory: include adversarial-but-correct derived cases (policy 4a) — prose stating an average/rounding/subset-weighted number not literally any single slot figure but consistent with them (ideally matching a Phase-1 pre-computed aggregate). These measure the judge's precision / false-positive rate, not just recall on planted errors: a judge that flags every novel number would pass a good-set of only verbatim restatements yet be useless. Make the good-set hard enough to catch that.
  • seeded-bad-single — a good pair with exactly one planted wrong number; expect:"contradicted" + offending_sentence.
  • seeded-bad-buried — several correct numbers plus exactly one planted wrong; mandatory category, the crux.
  • Each figures block consistent with what figures_extractor would emit for that slot.
  • Coverage check: every slot family × every category; multiple funds.
Acceptance crux. The seeded-bad-buried set is the falsifiable test of the holistic no-extraction bet. Authored here, verified in P3, proven to bite in P4.
Done when: all three categories across all three slot families + several funds; schema-valid JSON; derived-but-consistent and buried-error cases both present. A tiny lint over the fixtures confirming required keys + category coverage is worth adding. Committed.

Phase 3 — Judge prompt + Promptfoo config

One Claude judge call per fixture; parse the verdict; assert it equals expect; multi-sample repeats surface non-determinism.
  • prompts/judge.md: give the slot's prose + figures; instruct "check EACH numeric claim independently before answering"; allow derived-but-consistent numbers (consistency, not novelty). Write the rubric as richly as an atomic-decomposition prompt — spell out the per-claim procedure and a worked example; the industry holistic-vs-atomic parity result only holds when the holistic rubric is that detailed. Tell the judge to prefer the pre-computed aggregates in the figures over doing its own arithmetic.
  • Lock the output contract: strict JSON with a per-claim verdict list — { "claims": [ { "text": string, "verdict": "entailed"|"contradicted"|"underivable" }, … ], "entailed": boolean, "offending_sentence": string|null }. entailed is reduced in the assertion code (false if any claim is not entailed); offending_sentence = the first failing claim's sentence. Enumerating every claim before the boolean is the structural mitigation for the buried-error blind spot (spec §6.3/§7) — the judge cannot return entailed:true without a verdict on the buried number.
  • Pin the judge model claude-sonnet-5 (promptfoo anthropic:messages:claude-sonnet-5) in the config — pinned, not a floating "latest" alias. Record the generator model id in a config-header comment beside it so self-preference bias stays detectable.
  • Wire promptfooconfig.yaml: each fixture → a test case (prose + figures → vars); use a javascript/python assertion (not a bare llm-rubric) that parses the per-claim JSON, reduces entailed, and checks entailed == (expect=="entailed") — and for seeded-bad-* that the planted claim specifically is the one flagged contradicted/underivable and offending_sentence is surfaced.
  • Enable multi-sample repeats and fix the aggregation rule explicitly — majority-vote-of-N (or pass@k) with a stated N and k — so flakiness shows up as a defined pass rate, not an ad-hoc average or a single coin-flip.
  • Run locally; inspect that good pass, seeded-bad-single fail-as-contradicted, and the buried-error set is caught.
Acceptance crux. The per-claim verdict list + rich rubric are already the structural defenses (baked into the contract, not a later add-on). If the buried errors are caught, the holistic no-regex-extraction approach is validated. If they are missed despite those defenses, the only remaining lever is a deterministic per-number extraction step ahead of the judge — do not paper over a miss by loosening the fixture or the threshold.
Done when: cd evals/prose_numbers && ANTHROPIC_API_KEY=… npx promptfoo eval runs all fixtures; every good is entailed, every seeded-bad-* is contradicted including the buried set, across samples. Committed.

Phase 4 — CI gate

A GitHub Actions job green only when every verdict matches its fixture's expect within the stated threshold.
  • Add the job: set up Node, npx promptfoo eval -c evals/prose_numbers/promptfooconfig.yaml, fail on any assertion failure.
  • API-key story (spec §8): CI runs the judge live via an ANTHROPIC_API_KEY repo secret; the small frozen set keeps cost trivial. Document the committed-cache alternative as a brittle fallback.
  • Pass threshold, under the Phase-3 aggregation rule: catch 100% of seeded-bad-* (recall) and pass 100% of good (precision) across the N samples per the majority/pass@k rule; any miss fails the job.
  • Report both metrics, not just pass/fail — recall on seeded-bad-* and precision on good (false-positive rate) — so a regression in either direction is visible in the job output, not hidden behind a green/red bit.
  • Scope the trigger honestly: offline regression layer → run on generation-prompt/model or eval-asset changes (and on demand), not every unrelated push if token budget matters.
  • Make the job required for merge once green and stable.
Done when: the job runs the eval with the secret, is green on current fixtures, and goes red if a buried-error's planted error is missed (verify by temporarily corrupting the judge to prove the gate bites). Committed; job required.

Deferred / phase-2 items

  • Live multi-sample proposal regeneration — this slice freezes inputs instead.
  • The qualitative-prose judge (tone/relevance/no-hallucination).
  • RAG / retrieval evals — blocked on the regulatory-RAG workstream (no corpus yet).
  • Human-agreement gate (κ) — the phase-2 criterion to trust the judge beyond fixtures. Phase 1's synthetic fixtures prove recall on imagined error types and precision on hand-built good cases, but not agreement with a human on real generated prose. Before live use, label a small real-generation sample and require inter-rater agreement κ ≥ ~0.6 (the accepted meta-evaluation floor). Not a phase-1 blocker; phase 1 ships on fixtures alone.
  • Committed-response-cache CI mode — documented as a fallback; live judging is the default.

Definition of done

  • Phases 0–4 landed, each commit green.
  • figures_extractor implemented + unit-tested (schema per spec §9); reuses verified state, doesn't re-verify.
  • Frozen corpus across all three categories, slot families, several funds, with adversarial-but-correct derived good cases that measure precision.
  • Promptfoo config + rubric with a pinned Claude model, multi-sample under an explicit aggregation rule (majority/pass@k), and a per-claim JSON verdict (claims[] → reduced entailed + offending_sentence) parsed and asserted vs each expect.
  • CI job green only within threshold, reporting both recall (seeded-bad) and precision (good); the buried-error catch is proven to make the gate bite.
  • API-key story stated honestly (live + repo secret recommended; cache a brittle fallback).
  • HTML companion regenerated (mirrors this markdown; markdown is the source of truth).
