Goal Ship ENH-10: an LLM-as-judge for prose-embedded numbers, driven by Promptfoo, gated in CI over a frozen corpus that proves the judge itself.
Stack Python (extractor + test) · Node/npx Promptfoo · a Claude judge model · GitHub Actions.
Four units, four phases:
P0 scaffold→ P1 figures_extractor→ P2 fixture corpus→ P3 judge + promptfoo→ P4 CI gate
Offline regression layer, run on prompt/model change — the runtime validate → repair loop stays the per-run guard for structured values.

Global constraints

  • The judge is a real LLM call. Fixtures-first freezes the eval's inputs, not the judge — running the suite needs a model API key. Do not build it as "no API key needed."
  • The 1:1 mapping is load-bearing: one fixture ↔ one Promptfoo test case ↔ one judge call ↔ one slot instance. Never per-proposal, never per-fund-type.
  • figures_extractor reuses verified state as ground truth — it does not re-verify it. State correctness is owned by test_proposal_validation.py.
  • Offline-only over tracked outputs. Nothing here regenerates a committed example. The extractor reads a ConsultantState from a fixture or a throwaway fake-LLM run.
  • The buried-error fixtures are the acceptance crux (P2 + P4). If the judge can't catch them, the holistic no-extraction bet is falsified — the fallback is stated, not silently pinned away.
  • HTML companion regenerated whenever this markdown changes. Conventional commits, commit after every green phase.

Phase 0 — Scaffold & toolchain

Stand up the evals/prose_numbers/ layout and make Promptfoo runnable locally with one command, before any judge logic exists.
  • Create evals/prose_numbers/{fixtures,prompts}/ + README.md.
  • package.json with promptfoo dev-dependency + script "eval": "promptfoo eval -c promptfooconfig.yaml"; prefer npx promptfoo (no global install).
  • Root make eval-prose-numbers (or npm script) that runs the eval.
  • README.md: local run, ANTHROPIC_API_KEY requirement, the "offline regression layer, not a per-run guard" framing.
  • Placeholder promptfooconfig.yaml with zero tests so npx promptfoo eval exits cleanly.
Done when: make eval-prose-numbers runs to a clean exit with no tests defined; README documents the local run + API-key requirement. Committed.

Phase 1 — figures_extractor

A pure, unit-tested state → figures function exposing only the judge-relevant deterministic numbers.
  • Decide the extractor's home (record it in the file header); prefer engine-side if it must import engine types.
  • Define the exact figures schema (flat dict): per-fund cfs, rank, weighted_alpha, allocation_pct, per-fund exposure_pct; portfolio-level weighted-avg alpha, weighted CFS, asset-class/geo exposure totals, count/share beating benchmark.
  • Implement extract_figures(state) -> dict — pure, no I/O; read from cfs_scores, portfolio, exposure compute.
  • Write tests/evals/test_figures_extractor.py: constructed ConsultantState → assert schema fields + values; assert purity (no input mutation).
  • Docstring states explicitly: reuses verified state as ground truth; does not re-verify against the workbook.
Done when: CONSULTANT_ENGINE_FAKE_LLM=1 .venv/bin/python -m pytest tests/evals/test_figures_extractor.py -v green; schema fields are exactly those a prose judge could be checked against. Committed.

Phase 2 — Fixture corpus

A frozen, hand-authored corpus that proves the judge across all three categories — the buried-error set is the crux.
  • Document the JSON schema: { slot_key, figures, prose, expect, offending_sentence?, category }.
  • good — real correct pairs across why.*/watch.*/macro.impact, several funds. Include ≥2 derived-but-consistent cases (policy 4a) so the judge is proven not over-strict.
  • seeded-bad-single — a good pair with exactly one planted wrong number; expect:"contradicted" + offending_sentence.
  • seeded-bad-buried — several correct numbers plus exactly one planted wrong; mandatory category, the crux.
  • Each figures block consistent with what figures_extractor would emit for that slot.
  • Coverage check: every slot family × every category; multiple funds.
Acceptance crux. The seeded-bad-buried set is the falsifiable test of the holistic no-extraction bet. Authored here, verified in P3, proven to bite in P4.
Done when: all three categories across all three slot families + several funds; schema-valid JSON; derived-but-consistent and buried-error cases both present. A tiny lint over the fixtures confirming required keys + category coverage is worth adding. Committed.

Phase 3 — Judge prompt + Promptfoo config

One Claude judge call per fixture; parse the verdict; assert it equals expect; multi-sample repeats surface non-determinism.
  • prompts/judge.md: give the slot's prose + figures; instruct "check EACH numeric claim independently before answering"; allow derived-but-consistent numbers (consistency, not novelty).
  • Lock the output contract: strict JSON { "entailed": boolean, "offending_sentence": string|null }, nothing else.
  • Pick and pin a concrete Claude judge model (a current Sonnet-class id) in the config.
  • Wire promptfooconfig.yaml: each fixture → a test case (prose + figures → vars); prefer a javascript/python assertion parsing the judge JSON: entailed == (expect=="entailed"), and for seeded-bad-* that offending_sentence is non-null. llm-rubric is the fallback.
  • Enable multi-sample repeats so flakiness shows up as a pass rate.
  • Run locally; inspect that good pass, seeded-bad-single fail-as-contradicted, and the buried-error set is caught.
If the buried errors are missed: first tighten the rubric (strengthen the "check each claim" instruction / add a worked example); if still missed, escalate to a per-number extraction step. Do not paper over a miss by loosening the fixture.
Done when: cd evals/prose_numbers && ANTHROPIC_API_KEY=… npx promptfoo eval runs all fixtures; every good is entailed, every seeded-bad-* is contradicted including the buried set, across samples. Committed.

Phase 4 — CI gate

A GitHub Actions job green only when every verdict matches its fixture's expect within the stated threshold.
  • Add the job: set up Node, npx promptfoo eval -c evals/prose_numbers/promptfooconfig.yaml, fail on any assertion failure.
  • API-key story (spec §8): CI runs the judge live via an ANTHROPIC_API_KEY repo secret; the small frozen set keeps cost trivial. Document the committed-cache alternative as a brittle fallback.
  • Pass threshold: catch 100% of seeded-bad-*, pass 100% of good across N samples; any miss fails the job.
  • Scope the trigger honestly: offline regression layer → run on generation-prompt/model or eval-asset changes (and on demand), not every unrelated push if token budget matters.
  • Make the job required for merge once green and stable.
Done when: the job runs the eval with the secret, is green on current fixtures, and goes red if a buried-error's planted error is missed (verify by temporarily corrupting the judge to prove the gate bites). Committed; job required.

Deferred / phase-2 items

  • Live multi-sample proposal regeneration — this slice freezes inputs instead.
  • The qualitative-prose judge (tone/relevance/no-hallucination).
  • RAG / retrieval evals — blocked on the regulatory-RAG workstream (no corpus yet).
  • Human-label calibration set — not needed for prose-number entailment.
  • Committed-response-cache CI mode — documented as a fallback; live judging is the default.

Definition of done

  • Phases 0–4 landed, each commit green.
  • figures_extractor implemented + unit-tested (schema per spec §9); reuses verified state, doesn't re-verify.
  • Frozen corpus across all three categories, slot families, several funds, incl. derived-but-consistent good cases.
  • Promptfoo config + rubric with a pinned Claude model, multi-sample, parsed {entailed, offending_sentence} asserted vs each expect.
  • CI job green only within threshold; the buried-error catch is proven to make the gate bite.
  • API-key story stated honestly (live + repo secret recommended; cache a brittle fallback).
  • HTML companion regenerated (mirrors this markdown; markdown is the source of truth).
