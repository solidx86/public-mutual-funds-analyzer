Pipeline Determinism & Eval Map

Draft — for every component: what feeds in, is it deterministic, and how (if at all) is it tested. Mirrors docs/review/pipeline-determinism-map.draft.md.

How to read this — the arc
① AS-IS what runs today + its gaps → ② TARGET 3 lanes · SQLite SSoT · deterministic screening (relabel retired) · one LangGraph island left — the consultant's → ③ EVAL deterministic today, LLM-judge planned
One idea under all three: Python owns numbers; the LLM owns categories & adjectives — every determinism & test decision follows from which side of that line a thing sits. (2026-07-06 decision: on the screening side even categories move to Python — the relabel is retired.)
Both pipelines start messy and non-deterministic — PDF-layout heuristics, live web scrapes, LLM labels & prose — and are deliberately funnelled into deterministic Python that owns every number.
non-det INGEST→ deterministic CORE→ fenced LLM  (categories + prose only — number slots pre-filled)
Design rule: Python owns numbers · the LLM owns categories & adjectives. In both pipelines the LLM is fenced to non-numeric work, and a deterministic test guards the fence.
Refinement (2026-07-06 decision): the screening side no longer needs the LLM at all — two of its four relabel fields come from a stated PHS field (171/171 deterministic), the other two are synthesized labels no consumer reads (dropped). In the target the line sharpens to: Python owns everything factual (numbers and categories); the LLM owns only adjectives — consultant prose.

Legend

DET — deterministic Python (regex parse, coordinate parse, math, render). Same input → same output; testable by recompute.
NON-DET (ingest) — non-deterministic by source: PDF-layout heuristics, live network, date.today().
LLM — non-deterministic by model: Step 1.5 categorical relabel (enum-bounded) or consultant prose. Never writes a number.
HITL — human-in-the-loop gate; a person approves before flow continues.
DEAD — on disk but read by no code (Master Prospectus PDFs).
DATA — a file / cache / workbook artifact (not a process).

Two source-fidelity gaps (not internal-consistency):
  • Stages 1 & 2 have no test against their source. A PDF misread or bad scrape becomes a wrong number in mfr_results.json — and every downstream recompute test still passes, because they recompute from the corrupted value. Garbage-in, self-consistent-garbage-out.
  • Step 1.5's number-safety is only conventional. test_relabel.py proves the 4 fields hold valid enum values, but nothing hard-stops the LLM from also editing a numeric field mid-run. "NEVER modify performance/holdings" is enforced by discipline, not a gate. ▶ The target resolves this by deletion — the Decision removes the LLM step, so no un-gated model write remains.
→ This is the case for the Target Architecture (= ENH-1/ENH-2 + ATH decoupling): one owned MFR lane with a typed schema + source-fidelity gate writing to the SQLite SSoT, instead of numeric extraction and the LLM relabel both editing a raw JSON blob.

Target — Strategy A · three decoupled lanes, SQLite SSoT  design target · not built

Rescopes Pipeline 1 (+ ENH-1/ENH-2, + ATH decoupling). Orchestration = Strategy A ("islands only"): LangGraph only where there's a cycle/HITL; everything else is plain Python or a scheduled job. The database is a SQLite file in-repo and is the single source of truth — every producer writes it, every consumer reads it.

Decision (2026-07-06) — retire the Step 1.5 relabel; drop asset_class + geography.

Remove the in-session LLM relabel entirely, and drop asset_class + geography as fields. Keep objective_class (col E) and phs_fund_type (col D) as plain deterministic PHS-sourced fields — no LLM. Sequence this jointly with the MFR→SQLite→Excel rework below, not as an isolated edit to extract_mfr.py.

Two facts kill the relabel's value: (1) all 22 LLM edits land in the two fields no consumer reads, while the two fields that are consumed got zero edits; (2) asset_class/geography have no source cell to be correct against — they're synthesized from the MFR name + Lipper string, so the "truth" is itself a judgement (e.g. PFSF is labelled United States while its own extracted geo numbers are Taiwan-heavy). The PHS fields, by contrast, read a stated value and only need transcribe-and-map.

Architectural consequence — screening loses its LLM and its only LangGraph. With the relabel gone there is no LLM, cycle, or HITL anywhere in screening. So Lane A no longer needs to be a LangGraph island: all three lanes become plain deterministic Python + a scheduler. The fidelity_gate stays, but as a deterministic source-fidelity assertion, not an LLM-guarding router. The only LangGraph left in the whole system is the consultant's generate → validate → repair loop. The Step-1.5 number-safety gap doesn't get fixed — it ceases to exist.

Three lanes with no code dependency on each other — they meet only at the database. ATH is on its own schedule (a daily NAV refresh; MFR is a monthly event), so it must not sit inside the MFR pipeline.

Why the split sits exactly there. A recompute is a hard assertion — the right tool for a number the system already knows exactly. An LLM-judge is fuzzy, probabilistic judgement — the wrong tool for a hard number, but the only tool for free-text prose the deterministic validator structurally cannot see. So: every number with an HTML anchor is recomputed deterministically; the judge is reserved for the prose surface (and prose-embedded numbers with no anchor).
