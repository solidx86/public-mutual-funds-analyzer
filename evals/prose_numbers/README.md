# Prose-number entailment eval (ENH-10)

Promptfoo harness that checks whether prose generated by `consultant_engine`
correctly restates the deterministic numbers it was given (or plausibly
derives from them), and flags contradictions — including a single wrong
number buried among several correct ones in the same passage.

This is an **offline regression layer** run on prompt/model change (and in
CI), not a per-run production guard. The runtime `validate → repair` loop
already inside `consultant_engine` is what gates structured values on every
real run; this suite exists to catch regressions in *prose* generation
against a frozen fixture corpus.

## Status: Phase 0 scaffold only

There are no fixtures, no judge rubric, and no real test cases yet — those
land in Phases 1–3 of the plan above. `promptfooconfig.yaml` currently only
proves the toolchain is wired: it points at a no-op `echo` provider with
zero test cases defined, so it needs no network call and no API key. That
changes once Phase 3 wires in the real judge model.

## Running it

From this directory:

```bash
npx promptfoo eval
```

Or the equivalent convenience target from the repo root:

```bash
make eval-prose-numbers
```

Both use `npx`, so no global `promptfoo` install is required and
`npm install` is optional — it's only needed if you want `promptfoo`
resolved from a local `node_modules` instead of npx's on-demand fetch.

## API key

**Once the real judge lands (Phase 3+), running this suite needs a model
API key.** The judge is a real LLM call — fixtures-first design freezes the
eval's *inputs*, not the judge itself — so this is explicitly **not** a
"no API key needed" tool. Set `ANTHROPIC_API_KEY` in your environment (see
the repo root's `.env.example`) to run it locally. CI supplies the key from
a repo secret (Phase 4).
